# Fable 5 邮件躺了三天，今天才敢点开

> **本站声明**：本文为独立第三方实测记录。作者是 Claude Pro 用户，与 Anthropic 无任何官方关联。本文内容不构成任何官方背书或投资建议。

---

Fable 5 的邮件躺在我邮箱里三天。

不是没看见。是不敢点。

## 一封没敢点的邮件

6月10号晚上，Anthropic 发了两个新模型——**Fable 5** 和 **Mythos 5**。我当晚写了一篇刷完公告的睡眠观察，那是站在外面的感受。

三天后，邮件来了。标题大概是「Your access to Fable 5 is now open」之类。我看了三遍，没动。

不是矫情，是真怕。

之前每次 Claude 大版本更新，我都兴冲冲升级，然后撞上一次半成品：API 不稳、终端掉线、上下文断了得重来。最狠一次是 Sonnet 4.5 上线当晚，我重写了三遍 skill 文件。

所以这封邮件，我让它躺了三天。

## 终端试了下，连 Opus 4.8 都没了

昨晚终于忍不住，在终端跑了一个老脚本。

报错：

```
Error: Model 'claude-opus-4-8' not found.
You may have access to claude-fable-5.
```

我盯着屏幕看了十秒。

Opus 4.8——我用了两个月的模型，从我工作流的核心，**消失了**。不是变慢了，是菜单里就没了。

我第一反应是：被偷家了。我给 Anthropic 付了 Pro 的钱，结果我用惯的工具被砍，给我塞一个我还没摸熟的新模型。

这种感觉你懂吗？就是你手机里的常用 App 突然被下架，给你装了一个看起来一模一样、但你完全不会用的新版本。

## 红迪上骂声一片

我跑去 Reddit 的 r/Claude，想看看是不是只有我一个人这样。

不用翻第二页。

置顶帖：「Fable 5 dropped, Opus 4.8 is gone, anyone else?」

评论里清一色：

- "Same here, terminal doesn't recognize it."
- "Anthropic should have grandfathered existing users."
- "I'm paying $20/month to beta test their new model."

我往下划了几十条。

**没有一条说"能用"。**

那一瞬间我觉得，这封邮件就是个乌龙。要么是 Anthropic 发错了，要么是给我开的权限有问题。

## 我差点就要放弃了

我打开 Claude app，准备关掉那个红色 banner 算了。

但手滑点到了左边的 tab。

不是 Chat，是 **Co-Work**。

Co-Work 是 Claude 桌面 app 里那个新模式——之前我没用过，听说能让 Claude 直接接管屏幕完成任务。

我切过去了。

屏幕上的输入框，比 Chat 那个简洁很多。没有模型选择，没有附件上传按钮。只有一行字：**Describe what you want done**。

我抱着死马当活马医的心态，把那个大任务的描述贴了进去。

按下回车。

**绿灯亮了。**

不是延迟的转圈，是真的进度条开始走。

我盯着看了三十秒——它在读我的文件、调用我的工具、按我之前写过的 skill 文档一步步往下走。

那一刻我承认——

**我爽到了。**

## Pro 用户也能真用

最让我意外的不是它能跑，是**它给我跑**。

我是 Pro 用户，$20/月那种。Fable 5 之前的所有顶级模型，Pro 账户要么用不到，要么只能跑轻量任务。

但这次，它真的在跑我那个大任务。

我之前给 Fable 5 配的 skill 和 SOP 文档——一个能跑几十步流程的自动化脚本——它在 Co-Work 里**真的在执行**。

我这才意识到：Anthropic 这次不是把模型换了个名字，是把"Pro 用户能不能跑长任务"这件事重做了。

邮件里有一行我之前没仔细看的话：**Available through June 22, 2026 as part of your plan.**

6月22日之前，我这个 Pro 套餐，都能跑。

## 我丢给了他一个大任务

昨晚我没睡。

不是因为紧张。是因为它**真的在跑**。

我那个大任务是——

（你懂的，写具体任务反而抢了明天测评的稿子。）

但简单说一句：我之前已经写好了一套 skill 流程，让 Claude 按照 SOP 一步步做。之前用 Opus 4.8 跑过几次，能成，但每一步都要我盯着，每隔几小时会断一次。

昨晚 Fable 5 一口气跑了 5 个小时，没断。

**5 个小时。**

对一个 Pro 用户来说，这等于我平时半个多月的额度全压在了一个任务上。

我做完这件事之后第一反应是——

「我之前那些加班的夜晚，是不是都可以这样？」

## 现在还不能出测评，但我可以先说几件事

按理说，我应该今晚就写一篇完整测评。但我没写。

原因很简单：Fable 5 在我这里才跑了 5 个小时。一次跑通不算数，得观察它在**真实连续多日任务**里会不会崩、会不会偷懒、会不会哪天突然变笨。

而且 6月22日是限时——如果到时候权限收回去，我就没东西可测了。

所以我给自己留了**一整个窗口期**。

但有几件事，我现在就能说：

**第一，邮件不要拖。**

我这三天不敢点开，结果错过了什么？什么都没错过。Fable 5 上线当晚就给我开通了，没晚一天早一天。但如果你也在等，我先告诉你：可以试了。

**第二，终端报错先别慌。**

Opus 4.8 真没了，菜单里没装 Fable 5 的本地客户端也会报"model not found"。这不是 bug，是 Anthropic 这次改版太猛，没给老用户过渡。

**第三，去 Claude app 的 Co-Work 模式。**

这是我昨晚最大的发现。终端跑不动不代表不能用，Co-Work 模式直接接管，长任务跑得动。

**第四，Pro 用户真的能跑大任务。**

这个我之前不敢想。Fable 5 给我打开了一个口子。

## 等我明天的测评

我那个大任务还在跑。

按它的进度，今晚应该能跑完第一轮，明早我会起来看结果。

如果它真的稳——

我之前那套 skill / SOP 工作流，可能要全部重新想一遍了。

不是优化，是**升级**。

明天的测评我会写：

- Fable 5 跑长任务到底稳不稳
- 5 小时额度重置机制
- Pro 用户能跑多大的项目
- 跟 Opus 4.8 比哪些地方变了
- 我那套 skill 流程在它身上要不要改

如果你也在等 Fable 5 的真实反馈——

**催更我。明晚见。**
